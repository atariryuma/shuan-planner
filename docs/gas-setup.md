# Google連携(GAS)セットアップ手順

所要時間: 約10分。Google Apps Script(無料)を自分のGoogleアカウントにデプロイすると、次の機能が使えます:

| 機能 | 場所 |
|---|---|
| 端末間のデータ同期 | データ → サーバーへ送信/取得 |
| Googleカレンダーから行事取り込み(複数カレンダー対応) | 週案 → 📆 行事を取得 |
| 週案を専用カレンダー「週案」へ書き出し | 週案 → その他 → 📤 |
| 週案・時数レポートのスプレッドシート出力 | 週案 → その他 → 📊 / データ → 📈 |
| 週案のメール提出(管理職宛) | 週案 → その他 → 📧 |
| Googleドライブへの自動バックアップ(20世代) | データ → 🗂 / 設定でON |

> データは**自分のGoogleドライブ内**に保存されます。第三者のサーバーは使いません。
>
> ⚠ **学校配付のGoogleアカウントについて**: 自治体のWorkspaceは管理者設定で外部アプリ・スクリプトの承認がブロックされていることが多く、その場合デプロイや承認ができません。**個人のGoogleアカウント**でのセットアップを推奨します。

## 1. スクリプトを作成する

1. [script.google.com](https://script.google.com) を開き「**新しいプロジェクト**」
2. エディタに表示されている `function myFunction() {...}` をすべて消し、
   このリポジトリの [`gas/Code.gs`](../gas/Code.gs) の中身を**全部貼り付け**る
3. 上部のプロジェクト名を「週案プランナー」などに変更して保存(Ctrl+S)

## 2. 同期トークンを設定する

1. 左メニューの歯車「**プロジェクトの設定**」→ 一番下の「**スクリプト プロパティ**」→「スクリプト プロパティを追加」
2. プロパティ: `TOKEN` / 値: **長いランダムな文字列**(例: パスワード生成ツールで32文字以上)
3. 「スクリプト プロパティを保存」

> このトークンがパスワード代わりです。後でアプリの設定画面に同じ値を入れます。

## 3. Webアプリとしてデプロイする

1. 右上「**デプロイ**」→「**新しいデプロイ**」
2. 歯車アイコン →「**ウェブアプリ**」を選択
3. 設定:
   - 説明: 任意(例: v1)
   - **次のユーザーとして実行: 自分**
   - **アクセスできるユーザー: 全員** ← ここを間違えると動きません
4. 「デプロイ」→ 初回は「アクセスを承認」が出るので自分のアカウントで許可
   (「このアプリはGoogleで確認されていません」と出たら「詳細」→「(プロジェクト名)に移動」で続行)
5. 表示された **`https://script.google.com/macros/s/…/exec`** のURLをコピー

## 4. アプリに設定する

週案プランナーの「**設定**」タブ → Google連携(GAS):

- **GAS WebアプリURL**: 手順3でコピーした `/exec` のURL
- **同期トークン**: 手順2で決めた値
- 「**接続テスト**」を押して ✅ が出れば完了

以後、「**データ**」タブの「☁⬆ サーバーへ送信 / ☁⬇ サーバーから取得」で同期、
「**週案**」タブの「📆 行事を取得」でカレンダーの予定を行事欄に取り込めます。

> **権限の承認について**: カレンダー・ドライブ・メールの機能を初めて使うときに「権限がありません」系のエラーが出る場合は、コード更新後の再承認が必要です。GASエディタで適当な関数(`doGet`など)を一度「実行」し、表示される承認画面で許可してください(カレンダー・ドライブ・メール送信の権限をまとめて承認できます)。

### 機能ごとのメモ

- **カレンダー書き出し(📤)**: 専用カレンダー「週案」が自動で作られ、授業がイベントとして登録されます。同じ週を再書き出しすると古いイベントは自動で置き換わります(自分で入れた他の予定には触れません)。スマホのGoogleカレンダーで「週案」の表示をONにすると、次の授業がすぐ確認できます。
- **メール提出(📧)**: 設定画面で提出先アドレスを入力してから使います。無料アカウントの送信枠は1日100通です。
- **スプレッドシート出力(📊/📈)**: マイドライブに「週案プランナー出力」が自動作成され、週ごと・年度ごとのシートが追加されます。
- **Driveバックアップ(🗂)**: マイドライブの「週案バックアップ」フォルダにJSONを保存し、最新20世代だけを残します。復元はファイルをダウンロードして「データ → インポート」へ。

## 5. スクリプトを更新するとき(重要)

コードを修正したら:

1. 「デプロイ」→「**デプロイを管理**」
2. 鉛筆アイコン(編集)→ バージョン:「**新バージョン**」→「デプロイ」

⚠ **「新しいデプロイ」を作ってはいけません**。URLが変わってしまい、アプリ側の設定が無効になります。

## トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| 接続テストで「CORSエラー」 | デプロイ設定が「アクセスできるユーザー: 全員」になっているか確認。/exec のURLか確認(/devは不可) |
| 「GASがHTMLを返しました」 | 未承認のままか、アクセス設定が「全員」以外。手順3-4をやり直す |
| 「認証エラー: トークンが一致しません」 | スクリプトプロパティ `TOKEN` とアプリ側の同期トークンが完全一致しているか確認 |
| 「サーバー側にTOKENが未設定です」 | 手順2のスクリプトプロパティが保存されているか確認 |
| 動いていたのに急に全滅 | 「新しいデプロイ」を作ってURLが変わった可能性。デプロイを管理→現行URLをアプリに再設定 |
| 反応が遅い(数秒) | GASの仕様です(1〜5秒程度は正常)。同期は手動ボタン方式なので普段の操作は影響を受けません |

## セキュリティに関する注意

- URLとトークンを知っている人は誰でもデータを読み書きできます。トークンは推測されにくい長い文字列にし、他人と共有しないでください
- 児童生徒の個人名などの個人情報は同期データに含めない運用を推奨します
- このスクリプトには「全データ削除」のようなAPIは実装されていません(被害最小化のため)
