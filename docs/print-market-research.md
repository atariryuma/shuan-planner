# 週案印刷の市場調査と設計方針

調査日: 2026-06-13

## 結論

一般的な小学校向け週案として最も受け入れられやすい既定様式は、A4縦1枚である。

- 横軸: 月曜から金曜(土曜は任意)
- 縦軸: 朝活動、1校時から6・7校時
- 各授業セル: 上段に教科、下段に単元・本時内容
- 上部: 年度、期間、第何週、学校・学年組・氏名、押印欄
- 下部: 週時数、累計、標準時数、残り、年間進捗
- 下部: 今週の重点、反省・次週への課題、必要に応じて管理職の指導欄
- 印刷表現: 白黒でも判読できる罫線中心の公文書調

横向きは、複式、土曜授業、7校時、長文を多く記載する場合のワイド様式として残す。

ただし、全国一律の単一様式があるわけではない。自治体・学校ごとに提出方法や記載欄が異なるため、
「標準的なA4縦」と「学校独自様式に合わせられる柔軟性」の両方が必要になる。

## 市場の構造

### 1. 紙の週案簿・教員手帳

強み:

- 開けばすぐ見渡せる
- 手書きで自由に追記できる
- 管理職の押印や赤字訂正と相性がよい
- 法的・校内運用上の記録として理解されやすい

弱み:

- 固定時間割の転記が毎週必要
- 時数集計を別に行う必要がある
- 検索、複製、端末共有ができない

少年写真新聞社の週間計画ノートはA4/B5とも1,210円で販売されており、予定の見通し、
ミス防止、業務改善を価値として訴求している。つまり、利用者は単なる罫線ではなく、
「一冊を見れば仕事が分かる」状態に対価を払っている。

### 2. Excelテンプレート・マクロ

強み:

- 既存の紙様式をそのまま再現しやすい
- 学校内で独自に変更しやすい
- 印刷結果への安心感がある

弱み:

- マクロ制限、OS差、ファイル破損、版管理がある
- 職員室PCに利用場所が固定されやすい
- 数式や参照範囲を壊しやすい
- 学年・年度更新の保守が利用者任せになる

代表的な公開例は「基本レイアウトの複製」と「週・月・学期の時数計算」を主価値としている。

### 3. Googleスプレッドシート・Apps Script

強み:

- 場所と端末を選ばず編集できる
- 固定時間割、データベース、時数集計を連携できる
- Google Classroomなど周辺サービスへ展開できる

弱み:

- 初回承認や学校アカウントの制限が導入障壁になる
- セル操作とシート切替がスマートフォンでは重い
- テンプレートの複製後に更新を配布しにくい

2025年公開のSpreadsheet週案簿は、固定時間割転記、週入力、時数計算、端末を選ばないこと、
Google Classroom連携を主要価値としている。2026年には利用者要望から朝学習の1/3時数対応版が
追加されており、モジュール時数は実需のある要件と判断できる。

### 4. 統合型校務支援

一般に学校全体のアカウント、名簿、成績、出欠、時間割等と連携できることが強みになる。
一方、個人の教師がすぐ試せず、自治体調達・管理者設定・研修が必要になるため、
個人導入できる週案専用ツールとは購買・導入経路が異なる。

本アプリは統合校務支援を置き換えるより、
「導入決裁なしで使え、提出物は従来様式のまま」という位置を狙うのが合理的である。

## 利用者セグメント

### 学級担任

最優先ニーズ:

- 固定時間割から今週を短時間で作る
- 行事・変更だけ直す
- 週時数と累計を間違えない
- A4縦1枚で管理職へ提出する
- 反省を翌週へつなげる

標準画面・標準印刷はこの層を基準に設計する。

### 専科・中学校教科担任

最優先ニーズ:

- 学級ごとの進度差を把握する
- 同じ教材をどの学級まで実施したか確認する
- 学級別の時数を出す

全学級の単純合算時数は標準時数と比較できないため、印刷では学級別を基本にすべきである。

### 複式学級

最優先ニーズ:

- 2学年を同じ時間枠に併記する
- 直接・間接指導を区別する
- 学年別に時数を集計する

情報量が多いため、A4縦を標準にしつつ、横ワイドを実用的な代替として残す。

### 管理職

最優先ニーズ:

- 次週の授業内容と進度を短時間で確認する
- 安全管理上の注意を見落とさない
- 反省や児童の状況を確認する
- 承認した内容と、その後の変更を区別する

管理職向けには装飾より、罫線、押印欄、変更履歴、未確認警告が重要になる。

## Jobs to be Done

中心となる仕事は「週案を作る」ではなく、次の連続した仕事である。

1. 基本時間割と年間指導計画から次週の案を作る
2. 行事、短縮日課、休業、進度差を反映する
3. 教科・単元・本時内容・安全上の注意を確認する
4. 週時数と累計が妥当か確認する
5. 従来の様式で提出・承認を受ける
6. 実施後の変更と反省を記録する
7. 次週・次年度の計画に再利用する

本アプリは1と4には強い。市場でさらに優位になるには、3、5、6を強化する必要がある。

## 利用者が購入・採用する理由

公開されている週案ツールの訴求は、概ね次の順で共通する。

1. 毎週の時間割転記をなくす
2. 時数計算の間違いをなくす
3. 年間進捗を見えるようにする
4. 職員室以外でも確認・編集できる
5. 既存の紙の週案と同じ形で提出できる

このため、独自性の高い画面よりも、提出物が従来様式に見えることが導入障壁を下げる。

## 市場での差別化方針

### 提供価値

「紙の週案と同じ安心感で、転記と時数計算をなくす」

### 強みとして維持するもの

- インストール不要
- オフラインでも利用可能
- 無料で個人導入可能
- 固定時間割、年間指導計画、時数、印刷が一体
- 小学校、中学校、専科、複式に対応
- Google連携は任意

### 市場に伝わりにくい機能

高機能であることを一覧にしても、初見の教師には負担に見える可能性がある。
通常画面では次の4段階に翻訳して訴求する。

1. 時間割を呼び出す
2. 今週の変更だけ直す
3. 時数を確認する
4. A4縦で提出する

モジュール、読み替え、複式、同期などは対象者別の詳細説明へ分離する。

## 週案が持つ記録・証跡の要件

元校長による公開事例では、週案に年間指導計画に基づく予定、ねらい、安全管理、週の反省、
児童への対応を記載し、管理職が確認・承認する運用が示されている。また、承認後の変更は
元の記載を消さず、別色で見え消し訂正する運用が紹介されている。

この実態から、将来的には次のデータを分ける必要がある。

- 承認時の計画
- 実際に実施した内容
- 中止・振替の理由
- 変更日時
- 安全管理上の注意
- 管理職コメントまたは確認状態

現状の「日付が過ぎた予定を実施済みとみなす」方式では証跡として弱い。

## 現行実装との差分

### 解消した差分

- 既定がA4横だった
- A4縦で、めあて・反省・時数表を横一列に詰めていた
- 週案下部の時数表が週・累計のみだった
- 設定画面が横向きを推奨していた

### 今後の差分

- 計画時数と実施時数を明示的に分ける
- 週案印刷前に未確認の授業を警告する
- 印刷プレビューをアプリ内に表示する
- 学校独自様式に合わせた欄名・並び順のプリセットを提供する
- 専科では学級別の時数表を出せるようにする
- 安全管理の注意を通常メモと分けて表示する
- 承認後の変更を履歴として残す
- 休み時間、放課後、会議、ToDoを任意表示する

## 印刷仕様

### 標準

- A4縦
- 余白8mm
- 1週1ページ
- 5曜日を標準、土曜は設定時のみ追加
- 文字サイズ9ptを基準とし、内容超過時のみ段階縮小
- 教科は太字と左罫線で識別し、背景色に依存しない
- 時数表はページ最下部で全幅表示
- 罫線と文字だけでも意味が通じる白黒対応
- 手書き追記ができる最低限の余白を残す

### 時数表

学級担任では次を表示する。

- 週
- 累計
- 標準
- 残り
- 進捗

専科は複数学級を単純合算すると標準時数との比較が誤解を招くため、標準・残り・進捗を表示せず、今後学級別出力へ拡張する。

複式は学年ごとに週・累計を分ける。標準・残り・進捗の追加は、横幅と読みやすさを確認して別途行う。

## 推奨ロードマップ

### P0 提出できる標準週案

- A4縦を既定にする
- 時数表を全幅表示する
- 週、累計、標準、残り、進捗を表示する
- 横向きをワイド様式として維持する

### P1 計画と実施の信頼性

- 各授業に計画・実施・中止・振替の状態を持たせる
- 金曜または週末に一括確認する
- 印刷時に未確認を明示する
- 実施時数は確認済み授業だけを数える

### P2 管理職提出と安全管理

- 安全注意欄
- 確認・承認状態
- 変更履歴
- 提出用PDFの固定スナップショット

### P3 学校別プリセット

- 標準週案
- 反省欄重視
- 管理職指導欄付き
- 専科学級別
- 複式ワイド
- 土曜・7校時

### P4 個人業務の一元化

- 休み時間・放課後
- 会議・出張
- ToDo
- 翌週へ繰り越すメモ
