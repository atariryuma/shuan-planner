# 週案プランナー 全体要件・プロダクト設計

更新日: 2026-06-13

## 1. 文書の目的

本書は、週案プランナーのプロダクト要件、情報設計、UX原則、データ設計、印刷設計、改善方針を一元管理する正式な設計文書である。

開発者、AIエージェント、レビュー担当者は、個別機能の局所最適ではなく、本書に記載した利用者の作業全体を基準に判断する。

仕様と実装が食い違う場合は、現状コードを確認した上で本書またはコードを更新し、食い違いを放置しない。

### 1.1 文書の読み方

- 設計原則、データ整合性、集計条件、印刷品質は守るべき要件である
- 「改善目標」「ロードマップ」は目標状態であり、現行実装にすべて反映済みとは限らない
- 現在の挙動はコード、`README.md`、実画面で確認する
- 新しいエージェントは `AGENTS.md`、本書、対象機能の関連文書の順に確認する

### 1.2 2026-06-13時点の現在地

- 印刷はA4縦を標準とし、週案本体、指導計画詳細、時数集計を出力できる
- 学級担任、専科、複式学級を区別して時数を集計する
- 週案画面はセルの簡潔/詳細表示を切替できる(学習活動・評価規準の表示密度)
- 授業編集は選択中の教科・学級をサマリー表示し、パレットは「変更」で開く
- 年間指導計画は一覧と全画面編集を分離し、選択単元の各時を縦型カードで編集する
- 設定画面はカテゴリ単位の単一表示に切り替えた

## 2. プロダクトビジョン

小・中学校の先生が、年間指導計画、基本時間割、学校行事を基に、一般的な週案を短時間で作成し、時数を正しく管理し、提出可能な印刷物を得られるアプリを提供する。

目指す状態は「多機能な入力システム」ではなく、次の一連の仕事が一つの場所で完結することである。

1. 次週の授業を作る
2. 単元と本時の内容を確認する
3. 行事や学級差に合わせて修正する
4. 時数と進度を確認する
5. 印刷またはPDFで提出する
6. 実施後に反省を残す
7. 次週と次年度へ再利用する

## 3. 成功指標

### 利用体験

- 初回設定後、通常週の作成から印刷まで5分以内を目標とする
- 主要操作は説明書なしでも名称と配置から推測できる
- 一覧から編集対象へ2操作以内で到達できる
- 同じ情報を年間計画と週案へ二重入力しない
- 保存状態、集計対象、印刷対象が利用者に明確である

### 品質

- 時数集計が担任、専科、複式の単位を混同しない
- 既存データを無断で失わない
- A4縦・横で文字切れ、罫線切れ、白紙ページを発生させない
- 画面幅が狭くても主要操作に横スクロールを要求しない
- 長文や多数授業は縮小だけに頼らず、要約、展開、ページ分割で処理する

## 4. 対象利用者

### 学級担任

- 全教科の週案を作る
- 教科別の本週、累計、標準、残り、進捗を確認する
- 一週間の流れと行事を優先して俯瞰する
- 印刷物は一般的な週案簿形式を期待する

### 専科・教科担任

- 同じ教科を複数学級で担当する
- 学級ごとの単元進度と時数差を管理する
- 同一教材をどの学級まで実施したか確認する
- 詳細印刷では同一単元の共通情報を重複させず、授業行で学級を識別する

### 複式学級担任

- 1コマに複数学年の授業を持つ
- 直接指導、間接指導、ガイド学習を区別する
- 時数と年間計画を学年別に管理する
- 合同授業は、教科、本時、活動、評価、備考、指導形態が同一の場合のみ統合する

### 管理職・教務

- 週案の期間、担当、時数、重点、反省を短時間で確認する
- 必要に応じて押印または指導・助言を記入する
- 紙またはPDFで安定した様式を求める

## 5. 利用者の主要ジョブ

### 初回

1. 学校種、担任形態、学年または担当学級を設定
2. 時程を確認
3. 基本時間割を登録
4. 年間指導計画を取り込む
5. 印刷ヘッダーと押印欄を設定

### 毎週

1. 基本時間割または期間自動作成で授業枠を作る
2. 行事、休業、変更を反映
3. 単元、本時、活動、評価を確認
4. 学級差や手動記載を調整
5. 時数と進度を確認
6. 印刷またはPDF保存
7. 実施後に反省と変更を記録

### 年度更新

1. 年度、学年、担当学級を更新
2. 必要な計画だけ残す
3. 前年度データをバックアップ
4. 新年度の標準時数、休業、学期区切りを確認

## 6. 情報アーキテクチャ

主ナビゲーションは次の5領域を維持する。

| 領域 | 主目的 | 一覧で優先する情報 |
|---|---|---|
| 週案 | 今週の作成・調整 | 日時、学級、教科、単元、本時 |
| 年間指導計画 | 年間・単元・各時の編集 | 教科、学年、単元、時数、入力状況 |
| 時数集計 | 時数・進度の確認 | 本週、実施、累計、標準、残り、見込み |
| 設定 | 学校・運用条件の変更 | 選択中カテゴリだけ |
| データ | 保存・同期・バックアップ | 保存先、最終保存日時、危険操作 |

### 表示密度の原則

- 一覧: 判断に必要な要約
- 展開領域: 補足情報
- 編集画面: 全項目
- 印刷本紙: 提出時に素早く読む要点
- 印刷詳細: 年間指導計画の全文

すべての情報を常時一覧へ表示してはならない。

## 7. 画面別要件

### 7.1 週案

#### 一覧セル

標準表示:

- 学級または学年
- 教科
- 単元名と単元内の時
- 本時のねらい

補助表示:

- 活動と評価は1行要約または表示切替
- 備考、中止、時数外、分数時数、指導形態は状態表示

要件:

- 一週間の俯瞰を阻害する長文を常時表示しない
- 「簡潔」「詳細」の表示密度切替を提供する
- 活動・評価を非表示にしても情報は失わない
- セル高が極端に増えた場合は全文を切り、編集画面で確認できるようにする

#### 授業編集

改善目標:

- 選択済みの教科と学級を先頭の1行で確認できる
- 教科・学級パレットは「変更」で開く
- 内容と備考へすぐ到達できる
- 年間指導計画の本時情報は要約表示
- 単元目標と3観点評価は折りたたむ
- 詳細設定は既定値から変更がある場合だけ開く

モーダルは1コマの局所編集に限定し、長い一覧編集には使わない。

### 7.2 年間指導計画

#### 計画一覧

- 教科、学年、教科書、単元数、総時数、入力充足率を表示
- 「印刷」「編集」「削除」の優先度を視覚的に分ける
- 削除は確認とUndoを併用する

#### 計画編集

現行の狭いモーダル内テーブルは段階的に廃止し、専用の全画面編集ビューへ移行する。

推奨構成:

- 左または上: 計画基本情報
- 中央: 単元一覧
- 右または下: 選択単元の編集
- 保存状態を常時表示

単元一覧の各行:

- 単元名
- 時数
- 各時入力数
- 単元目標・評価の入力状態
- 並べ替えハンドル
- 明示的なメニュー

小さな `↑ ↓ ×` を主要操作にしない。タッチ可能領域は原則44px相当を確保する。

#### 単元・各時編集

長文を5列の狭い表へ押し込まない。

1時間を1カードまたは1セクションとし、次を縦に編集する。

- 指導目標
- 学習活動
- 評価規準
- 観点

必要な機能:

- 前時の複製
- 次時の追加
- 並べ替え
- 複数時への一括入力
- 入力済み項目の概要
- キーボードで次項目・次時へ移動

### 7.3 時数集計

#### 学級担任

- 列: 教科
- 行: 本週、実施済み、累計、標準、残り、進捗

#### 専科

- 行: 学級と担当教科
- 列: 本週、実施済み、累計、標準、残り、進捗
- 学級ごとの標準時数を合算しない
- 0時間の担当学級も表示する

#### 複式

- 学年ごとに独立集計
- 合算値だけで判断させない

共通:

- 計画と実施を区別する
- 中止、時数外、分数時数、モジュール換算を一貫して反映する
- 月、学期、年度の境界は授業実日付で処理する

### 7.4 設定

カテゴリナビゲーションを押したら、選択カテゴリだけを表示する。

カテゴリ:

- 基本
- 担任形態
- 時程
- 年間・学期
- 表示
- 教科
- 印刷
- Google連携

禁止:

- 4列の狭いカードへ大量設定を並べる
- カード内部の横スクロール
- 横スクロールしないと追加・削除に届かない構造

推奨:

- 最大幅900～1100pxの1カラムまたは主従2カラム
- 表は画面幅に応じてカードリストへ切替
- 危険操作と通常設定を分離
- Google連携は初期状態で折りたたむ

### 7.5 データ

保存先を中心に再構成する。

1. この端末
   - 自動保存状態
   - エクスポート
   - インポート
2. Google
   - 最終同期
   - Googleへ保存
   - Googleから取得
   - ドライブバックアップ
3. レポート
   - 時数レポート
4. 危険な操作
   - 全データ削除

「Googleへ保存」「ドライブへバックアップ」「自動バックアップ」の違いを説明できる構造にする。

## 8. 年間指導計画と週案のデータ設計

### 基本原則

- 年間指導計画を正規データとする
- 自動反映内容は週案エントリへ複製保存しない
- 表示時に進度番号から該当授業を解決する
- 手動内容は上書き表示として保存する
- 手動内容へ切り替えても元の計画参照を保持する
- 中止時は当時の予定内容をスナップショットとして残す

### 計画

```text
Plan
  subjectKey
  grade
  textbook
  startOffset
  units[]
```

### 単元

```text
Unit
  id
  name
  hours
  goal
  criteria
    knowledge
    thinking
    attitude
  lessons[]
```

### 各時

```text
Lesson
  objective
  activity
  assessment
  viewpoint
```

### 週案エントリ

```text
Entry
  id
  subjectKey
  scope
  text
  auto
  note
  fraction
  advance
  noCount
  cancelled
  cancelledText
  guide
```

`scope` は専科では学級ID、複式では学年を表す。意味を混在させる変更は避ける。

## 9. 印刷要件

詳細は `docs/print-market-research.md` を参照する。

### 週案本紙

- 既定はA4縦
- 1週1ページ
- 日付、担当、押印欄、曜日、校時、行事、授業、重点、反省、時数を表示
- 活動・評価は短い要約
- 「活」「評・観点」は固定幅ラベルで本文位置を揃える
- 本紙へ全文を詰め込まない

### 指導計画詳細

- 本紙の後へ任意添付
- 同じ計画・単元は複数学級で共有する
- 対象学級を概要へ表示する
- 授業表に学級列を持つ
- 単元目標と3観点評価を学級ごとに反復しない
- `詳細 n/N` を表示する
- 長文量を考慮してページ分割する

### 時数表

- 担任: 教科別
- 専科: 学級別
- 複式: 学年別
- 本週、累計、標準、残り、進捗を表示

### 検証

- A4縦・横の実寸DOMまたはPDFで確認
- 全ページを画像化して確認
- ページ、表、固定高領域の縦横オーバーフローを計測
- 全セルの罫線を確認
- 詳細ページの過剰な下部余白を確認

## 10. UX原則

### 段階表示

利用者が今判断する情報だけを先に見せる。詳細は展開、編集、別ページへ分ける。

### 編集対象中心

画面の中心は設定項目ではなく、利用者が現在編集している週、単元、授業にする。

### 一覧と編集の分離

- 一覧は比較と選択
- 編集は入力と確認

長文を一覧表内で直接編集する設計は避ける。

### スクロール

- ページ全体の縦スクロールを基本とする
- 内部スクロールは短い選択肢一覧などに限定する
- 入れ子の縦スクロールを避ける
- カード内部の横スクロールは禁止する

### 操作サイズ

- 主要ボタンと危険操作は文字ラベルを持つ
- アイコンのみの操作にはアクセシブルネームと十分な領域を持たせる
- タッチ対象は原則44px相当

### 保存

- 自動保存状態を常時表示
- 保存が必要な編集画面では「キャンセル」「保存」を明確にする
- 「閉じる」と「保存」を混同しない

## 11. 技術設計

### 現状

- 素のHTML、CSS、JavaScript ES Modules
- ビルド工程なし
- `localStorage` にスキーマ付き状態を保存
- GitHub Pagesで配信
- Service WorkerによるPWAキャッシュ
- GASは任意連携
- 印刷は専用DOMと動的 `@page`

### モジュール責務

| モジュール | 責務 |
|---|---|
| `js/store.js` | 状態、移行、年間計画解決、時数計算 |
| `js/views/week.js` | 週案表示と編集 |
| `js/views/plans.js` | 年間指導計画 |
| `js/views/stats.js` | 時数と進度 |
| `js/views/settings.js` | 設定 |
| `js/views/data.js` | 保存、同期、バックアップ |
| `js/print.js` | 印刷DOM |
| `js/print-hours.js` | 役割別印刷時数モデル |
| `js/csv.js` | 年間計画の取込・出力 |

### 変更方針

- フレームワーク導入は目的にしない
- 大規模画面の分割が必要なら、まず表示関数と純粋モデルを分離する
- DOM非依存の計算は別モジュールへ置き、Nodeテスト可能にする
- データ構造変更は正規化と移行を追加する
- localStorage容量を意識し、自動生成情報を重複保存しない

## 12. 非機能要件

### 信頼性

- 保存失敗を通知する
- 破壊的操作はUndoまたは確認で保護する
- インポート前に件数を提示する
- 不正な学級IDや教科キーを無言で集計から落とさない

### 性能

- 数十週、複数学年、年間計画全件でも通常操作が待たされない
- 再描画時に不要な全年度計算を増やさない
- 印刷生成は複数週でも終了する

### アクセシビリティ

- フォーム要素にラベルを関連付ける
- 色だけで状態を表現しない
- キーボードで主要操作が可能
- 小さい補足文字へ必須情報を置かない
- コントラストを確保する

### プライバシー

- 児童生徒の個人名を前提にしない
- Google連携は任意
- データ送信先と保存先を明示する
- 合言葉や接続情報を画面・ログへ露出しない

## 13. 改善ロードマップ

### P0: 日常編集の負担軽減

1. 週案の簡潔・詳細表示切替
2. 授業編集の教科・学級パレット折りたたみ
3. 設定カテゴリの単一表示
4. カード内部横スクロールの撤廃

### P1: 年間指導計画の再設計

1. 全画面編集
2. 単元一覧と選択単元編集の分離
3. 各時を縦型カード化
4. 複製、一括入力、キーボード移動
5. 入力充足率と未入力箇所の可視化

### P2: データと運用

1. データ画面を保存先別に再構成
2. 同期競合の説明と解決
3. バックアップ履歴の可視化
4. 学校向け初期設定プリセット

### P3: 発展

- 管理職確認ワークフロー
- 学校独自印刷テンプレート
- 年間計画の対応列マッピング
- 実施記録と計画変更履歴の比較

## 14. 受入基準

### UI変更

- 主要な編集対象が画面上で明確
- 狭いカード内に横スクロールがない
- 長文入力欄が内容確認に十分な幅を持つ
- 保存と閉じるの意味が明確
- タッチとキーボードの操作を確認

### 集計変更

- 担任、専科、複式のテストを持つ
- 0件、未設定、中止、時数外、分数時数を確認
- 標準時数を誤って合算しない

### 印刷変更

- A4縦・横を確認
- 罫線切れなし
- 縦横オーバーフローなし
- 長文、多学級、複式を確認
- 不要な白紙と過剰なページ重複なし

### データ変更

- 既存データを読み込める
- 欠損項目を正規化できる
- エクスポート・インポートが往復する
- Service Workerのバージョンを更新する

## 15. 設計上の禁止事項

- 新機能を設定画面へ無秩序に追加する
- 文章編集を狭い多列表へ追加する
- すべての詳細を一覧へ常時表示する
- フォント縮小だけでオーバーフローを解決する
- 専科の学級別標準時数を単純合算する
- 複式の学年別計画を混合する
- 自動反映内容を週案へ大量に複製保存する
- 保存を伴う画面で「閉じる」だけを置く
- 実寸確認なしで印刷変更を公開する
- `sw.js` のバージョンを上げずに公開する
