# UIテキスト規約

画面の文字は道具の一部。読ませない・迷わせない・繰り返さない。

## 原則

1. **ボタンは体言止め2〜6字**(「印刷」「取り込み」「元に戻す」)。「〜する」「〜してください」は使わない
2. **見出し・ラベルに句読点なし**。説明はラベルに書かず ⓘ(infoHTML)へ
3. **トーストは結果報告のみ20字以内**(「前週をコピーしました」)。操作の教育文・括弧の補足は書かない。次の行動が必要なときだけactionボタンを付ける
4. **画面に常設する説明文(hint)は1パネル1行まで**。2行を超える知識は ⓘ か docs/ へ
5. **専門用語の封じ込め**: GAS・localStorage・トークン等の実装用語は設定の該当欄と docs/ のみ。画面の総称は「Google連携」
6. **同一概念は同一表記**(保存=「Googleへ保存」、学級・学年・組を混ぜない)
7. **日付は `6/9` 形式**、期間は `6/9 〜 6/13`。年は年度表示(第N週)で代替。日時は `6/9 14:30`(fmtMDHM)。外部制約がある場所(シート名の「/」禁止等)は「6月9日週」のように月日表記
8. **絵文字アイコンは1機能1個・主要アクション(常時見えるボタン)のみ**(🖨 印刷、📋 基本時間割、📆 行事、🖌 連続入力、📥 取り込み=年間計画のみ)。⋯メニュー内・トースト・見出し・確認文では使わない(メニューはグループ区切り線で整理する)
9. **確認ダイアログは破壊的操作のみ**。可能ならダイアログを置かず Undo(元に戻す)で守る。週単位以上の削除は確認+Undoの両方。「キャンセル」は何もしないこと(隠れ副作用の禁止)
10. **数字は算用数字**、単位は「コマ」「時間」「週」
11. **ラベルの括弧補足は4字以内の名詞のみ可**(「組(任意)」○ /「時程(校時)」×)。それ以上の説明は ⓘ へ
12. **モーダルのフッター規約**: 閉じるだけのモーダルは「閉じる」1つ。変更を伴うモーダルは「キャンセル」+「保存」(主ボタン)。「完了」「OK」は使わない

## ⓘ(ポップオーバー)の使いどころ

- タッチで読めない title 属性の代わり(必須情報を title に置かない)
- 初見で意味が取れない可能性のある専門欄(係数・標準時数・分数時数)
- 法令根拠・運用上の注意などの「知りたい人だけ読む」情報
- 必ず ui.js の infoHTML() を使う(`<span title>ⓘ</span>` の手書き禁止。タッチで開かない)

## 用語対訳表(画面で使う語に統一)

| 画面で使う語 | 使わない語(実装用語) |
|---|---|
| Google連携 | GAS、Apps Script |
| 接続先URL | WebアプリURL、デプロイURL |
| 合言葉 | 同期トークン、TOKEN(設定のⓘと docs/ のみ可) |
| Googleへ保存/取得 | サーバーへ送信/取得、push/pull |
| ドライブ | Drive |
| 時数集計・時数レポート | 時数表(同一機能に複数名を与えない) |
| エクスポート/インポート | 書き出し/読み込み(データタブのJSONファイル操作に限る) |
