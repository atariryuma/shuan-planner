# アプリ本体をGASで配信する（職場など、今のサイトに入れない場合）

GitHub Pages（`https://atariryuma.github.io/shuan-planner/`）が職場のフィルタで開けない、
といったときに、**まったく同じアプリを Google Apps Script の Web アプリとして配信**できます。
家＝今のサイト／職場＝GAS版、で **同じ合言葉（同期トークン）を入れておけば、
データは既存のクラウド同期で行き来**します（アプリ本体のソースは一切変えていません）。

> 失うもの: オフライン利用・ホーム画面インストール（PWA）。GASのiframe内では使えないため。
> 得るもの: GitHub不要・URL1つ・Google内で完結。

---

## 仕組み

- `gas/build-webapp.mjs` … 本体（`js/` の18モジュール＋CSS＋アイコン）を **1枚の `gas/Index.html`** に束ねる依存ゼロのビルダ。
- `gas/Index.html` … 生成物。GASプロジェクトに「Index」HTMLファイルとして貼るだけ。
- `gas/Code.gs` … `doGet` が **ページ配信** と **同期API** を兼ねる（同期はすべてPOSTなので衝突しない）。

更新したいとき:

```
node gas/build-webapp.mjs     # gas/Index.html を作り直す
```

→ GASの「Index」ファイルに貼り直し、デプロイを「新バージョン」で更新（URLは変わらない）。

---

## セットアップ（5分）

1. https://script.google.com で **新しいプロジェクト**（既に同期用のGASがあるなら、それと同じプロジェクトでよい）。
2. ファイルを2つ用意する:
   - **`Code.gs`** … `gas/Code.gs` の中身を貼る（同期API＋ページ配信）。
   - **`Index`（HTML）** … 左の「＋」→「HTML」で `Index` を作り、`gas/Index.html` の中身を貼る。
3. 「プロジェクトの設定」→「スクリプト プロパティ」に
   `TOKEN` = 任意の長いランダム文字列 を追加（アプリ設定の「同期の合言葉」と同じ値）。
4. 「デプロイ」→「新しいデプロイ」→ 種類「ウェブアプリ」
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**（自分だけでも可）
   でデプロイ。表示された **`/exec` で終わるURL** がアプリのアドレス。
5. そのURLを職場のブラウザで開けば、家と同じアプリが動く。
   設定画面の「クラウド同期」に **同じ `/exec` URL と同じ合言葉** を入れれば、家と職場でデータが同期する。

> 1つのデプロイでアプリ配信と同期APIを兼ねるので、URL・合言葉は **どちらも同じ** でOK。

---

## メモ

- `gas/Index.html` はビルド生成物。**手で編集しない**（次のビルドで上書きされる）。直すときは `js/` 側を直して再ビルド。
- 本体ソース（`js/ css/ index.html sw.js`）は無改変。GitHub Pages版はこれまで通り。
- Service Worker・sendBeacon はGAS環境では自動的に無効化される（コード側で `if`/`catch` 済み）。
