Google連携の設定手順

所要時間: 約10分。Google Apps Script(無料)を自分のGoogleアカウントに置くと、次の機能が使えます。

機能使う場所
端末間のデータ同期(自動同期も可)データ → Googleへ保存/取得
Googleカレンダーから行事の取り込み週案 → 📆 行事
週案を専用カレンダー「週案」へ書き出し週案 → ⋯ → カレンダーへ書き出し
週案・時数レポートのスプレッドシート出力週案 → ⋯ → シートへ書き出し / データ → 時数レポート出力
週案のメール提出(管理職宛)週案 → ⋯ → メールで提出
Googleドライブへのバックアップ(20世代)データ → ドライブへバックアップ / 設定で自動化
データは自分のGoogleドライブ内に保存されます。第三者のサーバーは使いません。
学校配付のGoogleアカウントについて: 自治体のWorkspaceは管理者設定で外部スクリプトの承認がブロックされていることが多く、その場合は手順3の承認ができません。個人のGoogleアカウントでのセットアップを推奨します。

1. スクリプトを作成する

  1. script.google.com を開き「新しいプロジェクト」
  2. エディタに表示されている function myFunction() {...} をすべて消し、 このアプリの gas/Code.gs の中身を全部貼り付ける
  3. 上部のプロジェクト名を「週案プランナー」などに変更して保存(Ctrl+S)

2. 合言葉(同期トークン)を設定する

  1. 左メニューの歯車「プロジェクトの設定」→ 一番下の「スクリプト プロパティ」→「スクリプト プロパティを追加」
  2. プロパティ: TOKEN / 値: 長いランダムな文字列(例: パスワード生成ツールで32文字以上)
  3. 「スクリプト プロパティを保存」
この値がパスワード代わりの「合言葉」です。あとでアプリの設定画面に同じ値を入れます。

3. Webアプリとしてデプロイする

  1. 右上「デプロイ」→「新しいデプロイ」
  2. 歯車アイコン →「ウェブアプリ」を選択
  3. 設定:
    • 説明: 任意(例: v1)
    • 次のユーザーとして実行: 自分
    • アクセスできるユーザー: 全員 ← ここを間違えると動きません
  4. 「デプロイ」→ 初回は「アクセスを承認」が出るので自分のアカウントで許可
    (「このアプリはGoogleで確認されていません」と出たら「詳細」→「(プロジェクト名)に移動」で続行)
  5. 表示された https://script.google.com/macros/s/…/exec のURLをコピー

4. アプリに設定する

週案プランナーの 設定 タブ → Google連携:

  • 接続先URL: 手順3でコピーした /exec のURL
  • 合言葉: 手順2で決めた値
  • 「接続テスト」を押して 「接続できました」 が出れば完了。続けて「自動同期をON」にすると、複数端末で自動的に同期されます
権限の承認について: カレンダー・ドライブ・メールの機能で「権限がありません」系のエラーが出る場合は、GASエディタで適当な関数(doGet など)を一度「実行」し、表示される承認画面で許可してください(必要な権限をまとめて承認できます)。

5. 2台目以降の端末を追加する(かんたん)

1台目の設定が済んだら、他の端末ではURLや合言葉を手入力する必要はありません。

  1. 設定済みの端末で 設定 → Google連携 →「他の端末を追加」を押す
  2. 表示された接続リンクをコピー(自分宛のメールやGoogle Keep・LINEのKeepメモに貼っておくとスマホで開けます)
  3. 新しい端末でそのリンクを開くだけ。自動で接続され、保存済みのデータが取り込まれ、自動同期もONになります
接続リンクには合言葉が含まれます。自分の端末だけで使い、他人と共有しないでください(リンクは開いた直後にアドレスバーから消えます)。

機能ごとのメモ

  • カレンダーへ書き出し: 専用カレンダー「週案」が自動で作られ、授業がイベントとして登録されます。同じ週を再書き出しすると古いイベントは置き換わります(自分で入れた他の予定には触れません)。
  • メールで提出: 設定画面で提出先アドレスを入力してから使います。無料アカウントの送信枠は1日100通です。
  • シートへ書き出し / 時数レポート出力: マイドライブに「週案プランナー出力」が自動作成され、週ごと・年度ごとのシートが追加されます。
  • ドライブへバックアップ: マイドライブの「週案バックアップ」フォルダにJSONを保存し、最新20世代だけを残します。復元はファイルをダウンロードして「データ → インポート」へ。

6. スクリプトを更新するとき(重要)

  1. 「デプロイ」→「デプロイを管理」
  2. 鉛筆アイコン(編集)→ バージョン:「新バージョン」→「デプロイ」
「新しいデプロイ」を作ってはいけません。URLが変わってしまい、アプリ側の設定が無効になります。

トラブルシューティング

症状原因と対処
接続テストで「CORSエラー」デプロイ設定が「アクセスできるユーザー: 全員」になっているか確認。/exec のURLか確認(/dev は不可)
「GASがHTMLを返しました」未承認のままか、アクセス設定が「全員」以外。手順3〜4をやり直す
「認証エラー: 合言葉が一致しません」スクリプトプロパティ TOKEN とアプリ側の合言葉が完全一致しているか確認
「サーバー側にTOKENが未設定です」手順2のスクリプトプロパティが保存されているか確認
動いていたのに急に全滅「新しいデプロイ」を作ってURLが変わった可能性。デプロイを管理 → 現行URLをアプリに再設定
反応が遅い(数秒)GASの仕様です(1〜5秒程度は正常)。普段の入力操作は影響を受けません

セキュリティに関する注意

  • URLと合言葉を知っている人は誰でもデータを読み書きできます。合言葉は推測されにくい長い文字列にし、他人と共有しないでください
  • 児童生徒の個人名などの個人情報は同期データに含めない運用を推奨します
  • このスクリプトには「全データ削除」のようなAPIは実装されていません(被害最小化のため)
